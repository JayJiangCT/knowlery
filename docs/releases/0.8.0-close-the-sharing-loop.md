# Knowlery 0.8.0 — Close the Sharing Loop: Design & Usage

> Theme: **close the sharing loop, pay the quality debt** — knowledge bundle *export*
> went headless (completing OKF end to end), the retrieval engine learned to give an
> honest "no", and two release cycles of engineering debt were cleared. Accepted
> specs: `specs/0.8.0/`.

---

## Part I — Positioning: what 0.8.0 completed

0.5.0 built the export review UI; 0.7.0 made the *receiving* side headless. 0.8.0
closed the remaining asymmetry: **the producing side works without Obsidian too.**
The blocker was never the compiler — it was the review gate ("nothing ships
unreviewed", 0.5.0's core safety property), which existed only as modal UI.

The design insight that unlocked it: **the review state already is a file.**
`.knowlery/export-scope.json` persists per-bundle seeds, hop depth, and per-item
review statuses with content hashes that auto-invalidate on edit. The modal was one
editor of that file; 0.8.0 gave it a second editor — the CLI — and the review state
became shared between shells: start reviewing in Obsidian, compile from a terminal,
or the reverse. For identical approvals the two shells produce **byte-identical**
bundles.

The second thrust: retrieval honesty. An agent acting on retrieval output cannot
distinguish "the vault knows this" from "one word collided" — so the engine learned
to say "no confident matches" precisely when that is the true answer.

## Part II — Technical design

### F1. Headless bundle export (`bundle export` / `review`)

- **Export-side core inversion**: `collect.ts` / `export-scope.ts` / `compile.ts`
  moved from `App` onto `VaultFs` plus a new **`LinkResolver`** abstraction — the one
  thing VaultFs cannot give (wikilink target resolution). Obsidian's implementation
  wraps `metadataCache` verbatim; the headless one resolves exact path → unique
  basename (ambiguous = miss) from a single scan pass, parity-tested against the
  mock metadata cache on the eval fixture vault. `activity-ledger` inverted too —
  required for byte-identical `log.md` projection across shells.
- **The review gate as a file protocol**: `export <seed>` walks links from the seed;
  any unreviewed item prints the checklist (id, kind, title, risk hints,
  new/changed markers) and **exits 1 — nothing compiles**. `review --approve/--flag
  <id>...` records enumerated decisions with content hashes. There is deliberately
  **no approve-all flag**: approving stays a per-item act, and the agent conduct
  (below) routes it through the human.
- **`knowlery-cli` skill (builtin #14)**: the full CLI command surface plus the
  export review conduct — present the checklist to the user *completely* (warnings
  verbatim), translate only the user's stated decisions into enumerated calls, never
  approve on the agent's own initiative, echo back what was recorded.

### F2. Score-quality abstention (eval-calibrated)

- **The failure mode**: the old rule abstained only when *nothing anywhere* matched
  a structured field — one common word ("roadmap") colliding with one description
  defeated it. Measured: 2/8 unanswerable accuracy on the expanded golden set.
- **The confidence gate**: a result set is confident iff the **top candidate** is —
  (1) structured hit + ≥½ term coverage, or (2) prose coverage ≥⅔ with evidence
  above a floor, or (3) *pure* source-graph reach (zero direct matches — the
  bilingual mechanism). Calibration found a spec bug: evidence boosts as an
  unconditional anchor leaked 4/5 collision cases; clause 3 was narrowed to "pure".
- **Eval discipline**: golden set grew 3 → 8 unanswerable collision shapes + 2
  answerable hard negatives; the pre-gate engine run was frozen as the answerable
  floor; the sweep table (every constant tried, every casualty) lives in the spec
  §7 addendum. Result: **unanswerable 8/8, zero answerable regression**;
  `unanswerableAccuracy: 1` and the per-category floor are CI-gated.
- **Known boundary** (real-vault acceptance): 2-term queries whose top candidate
  matches exactly one term sit on the `>= ½` edge and can leak ("斑马的移动端路线图").
  Tightening past 0.5 over-abstains legitimate half-coverage answers; the fix needs
  a specificity signal on the *unmatched* terms — backlogged with seed cases.

### F3. Release engineering hygiene

- **npm Trusted Publishing (OIDC)**: the token is gone. npmjs.com trusts
  `release.yml`'s workflow identity; provenance attestations generate automatically;
  the 0.7.0-era 2FA/token failure mode is structurally impossible. Setup:
  trusted publisher registered on npmjs.com (owner/repo/workflow/allowed-actions),
  `NPM_TOKEN` secret deleted, token revoked.
- **Idempotent publish**: a registry-existence check makes tag re-runs a green no-op.
- **CLI EPIPE handling**: `knowlery query | head -1` (the agent-truncates-output
  shape) exits 0 silently — one stream-level listener, verified counterfactually
  (removing it fails the smoke test) and with `pipefail` observing the node side.
- `repository` field added (the OIDC identity check needs it); `bin` path warning
  fixed.

### F4. Repo lint hygiene

- **Unscoped eslint crash fixed**: typed rules had applied to files without type
  info. The obsidianmd preset is scoped to `src`; `tests/` + `evals/` came under
  typed lint via `tsconfig.eslint.json`; `npm run lint` *is* `eslint .` now; all
  hidden directories ignored via one glob.
- **The debt was measured, then paid**: 8 disabled rules re-enabled at error —
  ~107 violations fixed (promise hygiene at every JSX boundary, `require`→import,
  typed `JSON.parse` everywhere).
- **Settings tab modernized with a fallback**: Obsidian 1.13 deprecates
  `display()` for declarative `getSettingDefinitions()` — but 1.13 turned out to be
  **Catalyst-only** (found at acceptance: blank settings page on public 1.12.x).
  Final design: *one definitions array, two renderers* — the framework renders it
  declaratively on 1.13+ (settings search included), a ~60-line interpreter renders
  the same definitions imperatively below; `minAppVersion` stays 1.12.2. When 1.13
  goes public, deleting the fallback is a two-line change.
- **CI now exists**: `ci.yml` runs lint + the full suite on every PR — previously
  nothing did.

### Numbers

Tests 247 → 281 · unanswerable 2/3 → 8/8 · lint: from fake-zero (rules off) to
true-zero (repo-wide, error severity) · long-lived credentials: 1 → 0.

## Part III — Usage guide

### Share a topic entirely from the terminal

```bash
knowlery bundle export drone-delivery            # prints checklist, exit 1 if unreviewed
knowlery bundle review drone-delivery --list --json   # fetch checklist (agents parse this)
knowlery bundle review drone-delivery --approve concepts/drone-delivery concepts/flight-safety
knowlery bundle export drone-delivery --zip      # compiles + zips once fully reviewed
```

The zip installs anywhere via `knowlery bundle install` (0.7.0). Review in either
shell — same scope file; hash invalidation means editing an approved page flips it
back to unreviewed automatically.

### Trust the "no"

`knowlery query` (all three transports) now abstains on low-confidence collisions:

```
$ knowlery query "mobile app roadmap"
No confident matches in this vault for: mobile, app, roadmap
```

Exit 0 — absence of knowledge is a result. If a question you *know* is answerable
abstains, the page's title/aliases likely don't share the question's words: add an
`aliases` entry (the `/cook` skill records nicknames and cross-language titles for
exactly this reason).

### FAQ

| Question | Answer |
|---|---|
| Can I approve everything at once? | No flag for it, by design. An agent may expand "approve all" into explicit ids **only after** showing you the full checklist. |
| Do CLI-exported and modal-exported bundles differ? | Byte-identical for the same approvals (tested). |
| Why did my query start saying "no confident matches"? | The 0.8 gate abstains when the best match covers too little of your question — previously you'd have received ranked noise. See the aliases advice above. |
| Does the plugin still work on Obsidian 1.12? | Yes — 1.12.2 remains the minimum; the settings tab carries a fallback renderer until 1.13 reaches public release. |
| What changed about releases? | npm publishing is token-free (OIDC) with automatic provenance, and re-running a release workflow is a safe no-op. |
